## 1. Overview

This document focuses specifically on the **handoff process** from Andent Web to PreFormServer - the critical boundary where prepared STL jobs transition from the web application's queue to the Formlabs print infrastructure.

### Key Principle

**Andent Web prepares. PreFormServer executes. Formlabs Dashboard tracks.**

Andent Web's responsibility:
1. **Intake & Classification** - Upload, classify, review cases
2. **Build Planning** - Group whole cases into Form 4BL manifests by compatible printer/resin/layer-height
3. **Handoff** - Send build manifests to PreFormServer with per-file preset hints
4. **Status Display** - Poll Formlabs Web API to display job status

PreFormServer handles: orient, pack, supports, dispatch to printers.
Formlabs Dashboard/Web API handles: cross-printer job tracking, screenshots, status updates.

### Current Repository Snapshot (2026-04-21)

- Implemented in code: compatibility-aware Form 4BL build manifests, job naming, scene creation/import/send calls, preset-hint propagation, auto-layout validation, whole-case rollback, row-level printer routing, `print_jobs` persistence, Print Queue endpoints/UI, Formlabs status sync, and screenshot caching.
- Verified in repository: `/api/uploads/classify`, `/api/uploads/rows/send-to-print`, planning preview, print-queue polling, screenshot caching, Form 4BL planner tests, and the full pytest suite (`187 passed` with plugin autoload disabled in this environment).
- Still not yet externally validated here: a live PreFormServer + Formlabs cloud round-trip against real services and printers.

### Planner Calibration Snapshot (2026-04-24)

- Fit decisions remain heuristic.
- Planner fit math now uses raw `x_mm * y_mm` for non-full-arch models.
- Geometry-detected Form 4BL full-arch models use a calibrated reduced effective footprint.
- Planner fit math no longer uses tooth support inflation.
- The Form 4BL evaluation benchmark now uses the live XY platform area (`353 mm x 196 mm = 69,188 mm^2`) as the Form 4BL planner budget. It improved from 11 planned builds to 4 planned builds on `From 4BL Test Data`, with model counts `[29, 34, 35, 24]` close to the Human Pack benchmark `[31, 36, 36, 22]`.
- The validation-only live rerun recorded before Human Pack budget calibration remains a publish-readiness risk because the current PreFormServer run failed both the calibrated and factor-1.0 control paths. Regenerate 4-build `.form` artifacts before final launch sign-off.

## 2. Handoff Flow

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    PHASE 1: PREFORMSERVER HANDOFF FLOW                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  STEP 1: BATCH CASES BY PRESET                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  Group Ready cases by preset:                                   │   │
│  │  • Ortho Solid → Batch_001                                      │   │
│  │  • Tooth → Batch_002 (auto-supports)                            │   │
│  │  • Die → Batch_003                                              │   │
│  │  Rule: One case cannot span multiple batches                    │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              ↓                                          │
│  STEP 2: CREATE PRINT JOBS                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  For each batch:                                                │   │
│  │  • Generate job name: YYMMDD-001, YYMMDD-002...                  │   │
│  │  • Create scene in PreFormServer                                │   │
│  │  • Import all case STLs into scene                              │   │
│  │  • Configure preset (supports, orientation)                     │   │
│  │  • Send to printer group                                        │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              ↓                                          │
│  STEP 3: TRACK IN PRINT QUEUE                                           │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  Andent Web Backend → Formlabs Web API                          │   │
│  │  • Poll every 5s for status updates                             │   │
│  │  • Cache: 5 second TTL                                          │   │
│  │  • Display: Screenshot, Job Name, Cases, Status                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              ↓                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  PreFormServer → Formlabs Printers                              │   │
│  │  • Orient & Pack                                                  │   │
│  │  • Generate supports (Tooth only)                                 │   │
│  │  • Print production                                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

## 3. PreFormServer API Contract

### Base Configuration

| Parameter | Value | Description |
|-----------|-------|-------------|
| Base URL | `http://localhost:44388` | Local PreFormServer instance |
| Protocol | HTTP/REST | JSON payloads, form-data for files |
| Authentication | None (local) | Assumes local trusted network |

### API Endpoints

#### 3.1 Create Scene

Creates a new dental scene for a patient case.

```http
POST /scene/
Content-Type: application/json

{
  "patient_id": "CASE12345",
  "case_name": "20260420_Case12345_Upper"
}
```

**Response:**
```json
{
  "scene_id": "scene_abc123",
  "status": "created",
  "patient_id": "CASE12345",
  "case_name": "20260420_Case12345_Upper"
}
```

#### 3.2 Import Model

Imports an STL file into an existing scene.

```http
POST /scene/{scene_id}/import-model
Content-Type: multipart/form-data

model: <binary STL file>
preset: "Ortho - Solid"  // Desired contract: optional hint for material settings
```

**Response:**
```json
{
  "model_id": "model_xyz789",
  "scene_id": "scene_abc123",
  "status": "imported",
  "dimensions": {"x": 65.5, "y": 55.2, "z": 12.3}
}
```

#### 3.3 Send to Printer

Submits a prepared scene to the printer queue.

```http
POST /print/
Content-Type: application/json

{
  "scene_id": "scene_abc123",
  "device_id": "printer_form3_001"  // Optional: specific printer
}
```

**Response:**
```json
{
  "print_id": "print_def456",
  "scene_id": "scene_abc123",
  "status": "queued",
  "queue_position": 3,
  "estimated_start": "2026-04-20T14:30:00Z"
}
```

#### 3.4 List Devices

Retrieves available printers.

```http
GET /devices/
```

**Response:**
```json
[
  {
    "device_id": "printer_form3_001",
    "name": "Form 3 - Station 1",
    "model": "Form 3",
    "status": "idle",
    "material": "Draft Resin V2"
  },
  {
    "device_id": "printer_form4_001",
    "name": "Form 4 - Station 2",
    "model": "Form 4",
    "status": "printing",
    "current_job": "print_abc789"
  }
]
```

#### 3.5 Get Scene Status

Checks the status of a scene in the print pipeline.

```http
GET /scene/{scene_id}
```

**Response:**
```json
{
  "scene_id": "scene_abc123",
  "status": "printing",  // created → queued → printing → completed/failed
  "print_id": "print_def456",
  "progress_percent": 45,
  "estimated_completion": "2026-04-20T16:45:00Z"
}
```

Current repository note: the implemented `PreFormClient.import_model()` now sends a mapped preset hint when the selected UI preset has a known PreFormServer translation.

Current repository note: rows are no longer grouped strictly by preset before handoff. Andent Web builds compatibility-aware Form 4BL manifests using whole-case planning; mixed presets may share a build only when they resolve to the same printer, resin, and layer-height family. Import into PreFormServer occurs by preset group inside one build so each file still receives its correct preset hint.

Current repository note: planner area estimates are heuristic. Non-full-arch rows use raw bounding-box area (`x_mm * y_mm`), while geometry-detected Form 4BL full-arch rows use a calibrated reduced effective footprint. Tooth support inflation is no longer part of planner fit math. Form 4BL uses the live `353 mm x 196 mm` XY platform area as its planner budget; Form 4B uses its physical `200 mm x 125 mm` XY platform area; unknown-printer and Form 4 fallback paths remain conservative.

### Current Build Manifest Contract (2026-04-21)

Andent Web now hands off planned `BuildManifest` objects rather than preset-only batches. Each manifest contains:

- `compatibility_key` for the printer/resin/layer-height family.
- ordered whole-case `case_ids`.
- ordered preset import groups with per-file `preform_hint` values.
- manifest JSON persisted on the print job for audit/debugging.

Rollback is also manifest-based: if PreFormServer validation rejects a candidate scene, Andent Web retries after removing the last-added whole case. If a single seed case still fails validation, the remaining rows are marked for manual review instead of splitting the case.

## 4. Client Implementation

### PreFormClient Class (Local API)

Location: [`app/services/preform_client.py`](../../app/services/preform_client.py)

```python
class PreFormClient:
    """Client for PreFormServer API interactions."""
    
    def __init__(self, base_url: str = "http://localhost:44388")
    def create_scene(self, patient_id: str, case_name: str) -> Dict[str, Any]
    def import_model(self, scene_id: str, stl_path: str) -> Dict[str, Any]
    def send_to_printer(self, scene_id: str, device_id: str) -> Dict[str, Any]
    def list_devices(self) -> List[Dict[str, Any]]
    def get_scene_status(self, scene_id: str) -> Dict[str, Any]
```

Current repository note: the implemented handoff path creates scenes, imports models with preset hints, runs auto-layout and scene validation, and routes to the row-selected printer when one is set; otherwise it falls back to `"default"`.

### FormlabsWebClient Class (Cloud API)

New client for Formlabs Web API:

```python
class FormlabsWebClient:
    """Client for Formlabs Web API interactions."""
    
    def __init__(self, api_token: str, base_url: str = "https://api.formlabs.com/v1")
    def list_print_jobs(self) -> List[Dict[str, Any]]
    def get_job_status(self, job_id: str) -> Dict[str, Any]
    def get_job_screenshot(self, job_id: str) -> bytes
    def get_printer_info(self, printer_id: str) -> Dict[str, Any]
```

### Usage Pattern

```python
from app.services.preform_client import PreFormClient
from app.services.formlabs_web_client import FormlabsWebClient

async def handoff_and_track(batch_id: str, stl_paths: List[str], preset: str):
    # Step 1: Handoff to PreFormServer
    with PreFormClient() as preform:
        scene = preform.create_scene(
            patient_id=batch_id,
            case_name=f"Batch_{batch_id}"
        )
        
        for stl_path in stl_paths:
            preform.import_model(scene["scene_id"], stl_path)
        
        print_job = preform.send_to_printer(scene["scene_id"])
    
    # Step 2: Track via Formlabs Web API
    web_client = FormlabsWebClient(api_token=os.getenv("FORMLABS_API_TOKEN"))
    status = web_client.get_job_status(print_job["print_id"])
    
    return {"scene_id": scene["scene_id"], "print_id": print_job["print_id"]}
```

## 5. Data Mapping

### Andent Web → PreFormServer

| Andent Web Field | PreFormServer Field | Notes |
|------------------|---------------------|-------|
| `case_id` | `patient_id` | Extracted from filename |
| `original_filename` | `case_name` | Human-readable identifier |
| `stored_file_path` | `stl_path` | Local filesystem path |
| `model_type` | Preset hint | Translated through the UI preset mapping at the PreFormServer boundary |
| `preset` | Print settings | Stored for batching and translated to a PreFormServer preset hint on import |

### Model Type → PreFormServer Preset (Phase 1 Final)

| Andent Model Type | PreFormServer Preset | Support Generation | Orientation | Material | Layer Height | Printer |
|-------------------|---------------------|-------------------|-------------|----------|--------------|---------|
| `Ortho - Solid` | `ortho_solid_v1` | None (lay flat) | Flat on platform | Precision Model Resin | 100µm | Form 4BL |
| `Ortho - Hollow` | `ortho_hollow_v1` | None (lay flat) | Flat on platform | Precision Model Resin | 100µm | Form 4BL |
| `Die` | `die_v1` | None (lay flat) | Flat on platform | Precision Model Resin | 100µm | Form 4BL |
| `Tooth` | `tooth_v1` | **Auto-generate** | Auto-oriented with supports | Precision Model Resin | 100µm | Form 4BL |

**Key Rules:**
- **Only Tooth models get auto-support generation** - All others lay flat directly on build platform
- **All presets use**: Precision Model Resin, 100µm layer height, Form 4BL printer
- **Preset = Model Type** in UI, translated at API boundary

## 6. Error Handling

### Connection Errors

| Scenario | Behavior | User Feedback |
|----------|----------|---------------|
| PreFormServer unreachable | Queue rows, show "PreFormServer Offline" | Toast notification |
| Timeout (30s) | Retry once, then fail | "Connection timeout - try again" |
| Invalid response | Log error, mark row for retry | "Print service error" |

### API Errors

| HTTP Status | Meaning | Action |
|-------------|---------|--------|
| 400 Bad Request | Invalid payload | Log details, skip row |
| 404 Not Found | Scene/model not found | Check PreFormServer state |
| 409 Conflict | Scene already printing | Update local status |
| 422 Unprocessable | STL validation failed | Flag row for review |
| 500 Internal Error | PreFormServer error | Retry with backoff |

### Retry Strategy

```python
MAX_RETRIES = 3
RETRY_DELAY = 5  # seconds

for attempt in range(MAX_RETRIES):
    try:
        response = client.create_scene(...)
        break
    except ConnectionError:
        if attempt < MAX_RETRIES - 1:
            time.sleep(RETRY_DELAY * (attempt + 1))
        else:
            raise HandoffFailed("PreFormServer unavailable")
```

## 7. Status Synchronization

### PreFormServer → Andent Web Status Mapping

| PreFormServer Status | Andent Web Status | Description |
|---------------------|-------------------|-------------|
| `created` | `Submitted` | Scene created, not yet queued |
| `queued` | `Submitted` | In print queue, waiting |
| `preparing` | `Submitted` | Orient/pack/support generation |
| `printing` | `Printing` | Active print job (maps to Print Queue tab) |
| `completed` | `Completed` | Print finished successfully |
| `failed` | `Failed` | Print error or validation failure |
| `cancelled` | `Failed` | User or system cancelled |
| `paused` | `Paused` | Print job paused |

**Note:** Once handoff occurs, status tracking moves to **Formlabs Web API** for detailed printer status.

## 8. Security Considerations

### Local Deployment Model

```
┌─────────────────────────────────────────────────────────┐
│                    LOCAL SERVER                          │
│  (Mac/PC connected to Formlabs printers via USB)        │
│                                                          │
│  ┌─────────────────┐  ┌─────────────────┐               │
│  │ Andent Web      │←→│ PreFormServer   │               │
│  │ (FastAPI:8090)  │  │ (localhost:44388)│              │
│  └─────────────────┘  └─────────────────┘               │
│           │                    │                        │
│           ▼                    ▼                        │
│  ┌─────────────────┐  ┌─────────────────┐               │
│  │ SQLite DB       │  │ Formlabs        │               │
│  │ + STL Storage   │  │ Printers (USB)  │               │
│  └─────────────────┘  └─────────────────┘               │
└─────────────────────────────────────────────────────────┘
```

### Security Model

- **No authentication** between Andent Web and PreFormServer (local only)
- **API Token authentication** for Formlabs Web API (stored securely)
- **Trusted network** assumption for dental lab deployment
- **File validation** on STL import (size limits, format checks)
- **No external exposure** - Andent Web and PreFormServer bind to localhost
