## 1. Overview

The classification system identifies dental STL files by **model type** to enable automatic preset assignment. It uses a hierarchical approach:

```
Filename Analysis → Artifact Classification → Structure Resolution → Model Type Mapping
```

**Design Philosophy**: Trust filename patterns first (high confidence), use geometry as fallback (medium/low confidence), flag ambiguous cases for human review.

## 3. Classification Pipeline

### 3.1 Step 1: Case ID Extraction

**Function**: `extract_case_id()` — [`core/andent_classification.py:139`](../../core/andent_classification.py#L139)

**Rules**:
1. Extract embedded 7-8 digit numeric IDs from filename stem
2. Exclude tokens that match date patterns (`YYYYMMDD` or `YYYY-MM-DD`)
3. Split by `_` and filter:
   - Minimum 3 characters
   - Must contain at least one digit
   - Not in reserved keywords list

**Reserved Case Tokens** (cannot be case IDs):
```python
_RESERVED_CASE_TOKENS = {
    "antag", "antagonist", "biteguard", "bitesplint", "guard",
    "lowerjaw", "model", "modelbase", "modeldie", "splint",
    "tooth", "upperjaw", "unsectionedmodel",
}
```

**Examples**:
| Filename | Extracted Case ID | Reason |
|----------|------------------|--------|
| `20260420_Case12345.stl` | `CASE12345` | Second token after date |
| `Case12345_2026-04-20.stl` | `CASE12345` | First token before date |
| `ABC123_XYZ789.stl` | `ABC123` | First valid token |
| `model.stl` | `None` | Reserved keyword |
| `file.txt` | `None` | No valid token |

### 3.2 Step 2: Artifact Classification

**Function**: `classify_artifact()` — [`core/andent_classification.py:171`](../../core/andent_classification.py#L171)

**Priority-Ordered Rules** (first match wins):

#### Priority 1: Filename Keywords

| Keyword Pattern | Artifact Type | Confidence | Reason Example |
|-----------------|---------------|------------|----------------|
| `bitesplint`, `biteguard`, `bite_guard`, `nightguard`, `splint` | `ARTIFACT_SPLINT` | `high` | "filename matched splint/bite-guard keywords" |
| `modeldie`, `die` (as word) | `ARTIFACT_MODEL_DIE` | `high` | "filename matched die/model-die keyword" |
| `tooth` (as word) | `ARTIFACT_TOOTH` | `high` | "filename matched tooth keyword" |
| `modelbase` | `ARTIFACT_MODEL_BASE` | `high` | "filename matched model base keyword" |
| `antag`, `antagonist` | `ARTIFACT_ANTAGONIST` | `high` | "filename matched antagonist keyword" |
| Legacy arch suffix (`_upper`, `_lower`, `_u`, `_l`) | `ARTIFACT_MODEL` | `high` | "filename matched legacy upper/lower arch suffix" |
| `unsectionedmodel`, `upperjaw`, `lowerjaw`, `implant`, `model` | `ARTIFACT_MODEL` | `high` | "filename matched model/arch keywords" |

#### Priority 2: Geometry Fallback

Only applied if no filename keyword matches.

**Requires STL dimensions** (from `get_stl_dimensions()`):

| Condition | Artifact Type | Confidence | Reason |
|-----------|---------------|------------|--------|
| `max(X,Y) ≤ 35mm` AND `Z ≤ 35mm` | `ARTIFACT_TOOTH` | `low` | "small geometry fallback matched likely tooth artifact" |
| `Z ≤ 8mm` AND `max(X,Y) ≥ 40mm` | `ARTIFACT_SPLINT` | `low` | "thin geometry fallback matched likely splint artifact" |
| `max(X,Y) ≥ 45mm` | `ARTIFACT_MODEL` | `low` | "large geometry fallback matched likely model artifact" |

#### Priority 3: Unable to Classify

| Condition | Result |
|-----------|--------|
| No keyword match AND no geometry match | `ARTIFACT_UNKNOWN`, triggers `review_required` |
| No case ID extracted | `review_required`, reason: "Could not derive a stable case identifier" |

### 3.3 Step 3: Structure Resolution (Ortho Models Only)

**Function**: `resolve_ortho_structure()` — [`core/andent_classification.py:456`](../../core/andent_classification.py#L456)

**Applies to**: `ARTIFACT_MODEL`, `ARTIFACT_ANTAGONIST`, `ARTIFACT_MODEL_BASE`

**Goal**: Determine if the model is **solid** or **hollow**.

#### 3.3.1 Measurements Required

1. **Fill Ratio** = `actual_volume / bounding_box_volume`
   - Actual volume: From STL mesh calculation
   - Bounding box: `X × Y × Z / 1000` (in mL)

2. **Wall Thickness Stats** (ray-casting sampling):
   - Sample 64 triangles weighted by surface area
   - Cast rays inward from triangle centroids
   - Measure distance to opposite surface
   - Calculate percentiles: P10, P50
   - Calculate fraction of samples < 5mm, < 3mm

#### 3.3.2 Thresholds

**Structure Detection Thresholds**:

```python
# Fill ratio boundaries
HOLLOW_MAX_FILL_RATIO = 0.28          # Below this = likely hollow
core/andent_classification.py:35

SOLID_MIN_FILL_RATIO = 0.32           # Above this = likely solid
core/andent_classification.py:36

# Thickness thresholds for hollow
HOLLOW_MAX_P10_MM = 3.0                 # 10th percentile wall thickness
core/andent_classification.py:37

HOLLOW_MAX_P50_MM = 4.5                 # 50th percentile wall thickness
core/andent_classification.py:38

HOLLOW_MIN_THIN_FRACTION_UNDER_5MM = 0.7  # Fraction of walls < 5mm
core/andent_classification.py:39

# Thickness thresholds for solid
SOLID_MIN_P10_MM = 3.5                  # 10th percentile wall thickness
core/andent_classification.py:40

SOLID_MIN_P50_MM = 5.5                  # 50th percentile wall thickness
core/andent_classification.py:41

SOLID_MAX_THIN_FRACTION_UNDER_5MM = 0.55  # Fraction of walls < 5mm
core/andent_classification.py:42

# Sampling parameters
THICKNESS_SAMPLE_BUDGET = 64            # Number of ray samples
core/andent_classification.py:31

THICKNESS_MIN_SAMPLE_COUNT = 24         # Minimum valid samples required
core/andent_classification.py:32

THICKNESS_MIN_VALID_SAMPLE_FRACTION = 0.35  # Minimum fraction of samples with valid hits
core/andent_classification.py:33

THICKNESS_MIN_HIT_DISTANCE_MM = 0.2     # Minimum probe distance (avoid self-intersection)
core/andent_classification.py:34
```

#### 3.3.3 Decision Logic

**Hollow Classification** (ALL must be true):
```
fill_ratio ≤ 0.28
AND thickness_p10 ≤ 3.0mm
AND thickness_p50 ≤ 4.5mm
AND thin_fraction_under_5mm ≥ 0.7
→ STRUCTURE_HOLLOW (high confidence)
```

**Solid Classification** (ALL must be true):
```
fill_ratio ≥ 0.32
AND thickness_p10 ≥ 3.5mm
AND thickness_p50 ≥ 5.5mm
AND thin_fraction_under_5mm ≤ 0.55
→ STRUCTURE_SOLID (high confidence)
```

**Review Required** (ANY true):
- Mesh has boundary edges (not watertight)
- Mesh has non-manifold edges
- Manifold edge fraction < 0.95
- Sample count < 24
- Valid sample fraction < 0.35
- Missing thickness percentiles
- Fill ratio and thickness evidence don't agree
→ STRUCTURE_REVIEW (low confidence)

### 3.5 Step 5: Confidence Derivation

**Function**: `derive_confidence()` — [`app/services/classification.py:92`](../../app/services/classification.py#L92)

**Confidence Levels**:

| Condition | Confidence | Reason |
|-----------|------------|--------|
| `upstream_review_required = True` | `low` | Artifact classification flagged for review |
| `model_type is None` | `low` | Could not map to Phase 0 model type |
| `classifier_confidence == "high"` AND `case_id exists` | `high` | Strong filename match + case ID |
| Otherwise | `medium` | Geometry fallback or missing case ID |

### 3.6 Step 6: Status Assignment

**Function**: `derive_status()` — [`app/services/classification.py:108`](../../app/services/classification.py#L108)

**Status Flow**:

```
┌─────────────────────────────────────────────────────────┐
│ Input: confidence, model_type, preset, flags           │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
            ┌─────────────────────────┐
            │ forced_submitted?       │
            └─────────────────────────┘
                │           │
            YES ▼           ▼ NO
        ┌──────────┐   ┌─────────────────────────┐
        │Submitted │   │ forced_duplicate?       │
        └──────────┘   └─────────────────────────┘
                          │           │
                      YES ▼           ▼ NO
                  ┌──────────┐   ┌──────────────────────────┐
                  │Duplicate │   │ manual_override AND       │
                  └──────────┘   │ model_type AND preset?    │
                                   └──────────────────────────┘
                                      │           │
                                  YES ▼           ▼ NO
                              ┌──────────┐   ┌──────────────────┐
                              │ Ready    │   │ confidence?      │
                              └──────────┘   └──────────────────┘
                                                │           │           │
                                           high ▼      medium ▼      low ▼
                                        ┌────────┐   ┌────────┐   ┌──────────────┐
                                        │ Ready  │   │ Check  │   │ Needs Review │
                                        └────────┘   └────────┘   └──────────────┘
```

**Statuses**:
- `Ready` — Can be sent to print (high confidence + complete data)
- `Check` — Medium confidence, should be reviewed
- `Needs Review` — Low confidence or missing critical data
- `Duplicate` — File hash matches existing row
- `Submitted` — Already sent to print queue

## 4. Review Triggers

**Automatic Review Required** when ANY of:

| Condition | Review Reason |
|-----------|---------------|
| `artifact.review_required = True` | From artifact classification |
| `model_type is None` | "Unable to map artifact into the approved Phase 0 model types" |
| `confidence != "high"` | "Model type inferred from weaker signals and should be checked" (medium) or "Unable to classify artifact confidently from filename or geometry" (low) |
| `case_id is None` | "Case ID could not be detected from filename" |
| Structure resolution returns `STRUCTURE_REVIEW` | Mesh topology or sampling issues |
